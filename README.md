# clean-code-study-notebooklm
Using AI to transform Clean Code concepts into structured learning: summaries, mind maps, and practical questions.

Este projeto foi desenvolvido como parte do desafio da Accenture + DIO no bootcamp de Python para Análise e Automação de Dados.
A proposta surgiu a partir do meu interesse em aprofundar meus conhecimentos em boas práticas de desenvolvimento, utilizando como base a obra Clean Code de Robert C. Martin, um material que considero essencial para qualquer desenvolvedor que deseja escrever código de forma profissional, legível e sustentável.
Mais do que estudar o conteúdo, busquei explorar o uso da Inteligência Artificial como aliada na construção de um sistema estruturado de aprendizado.

## Contexto e objetivos: 
Criei este projeto com o objetivo de transformar o estudo técnico em um processo mais estratégico e organizado.

A ideia foi utilizar IA para desenvolver habilidades como:
Estruturar conteúdos complexos de forma clara.
Gerar resumos didáticos e reutilizáveis.
Criar perguntas práticas para fixação.
Construir um material de estudo contínuo e escalável.
Utilizar engenharia de prompts.
Organização do conhecimento.

## Engenharia de Prompts:
Uma das partes mais ricas deste projeto foi entender, na prática, que trabalhar com IA não é apenas fazer perguntas, é aprender a pensar melhor sobre como perguntar. Ao longo da construção deste caderno baseado em Clean Code de Robert C. Martin, percebi que a qualidade das respostas está diretamente ligada à clareza e intenção dos prompts. No início, minhas perguntas eram mais abertas, guiadas pela curiosidade:

"Quais principais objetivos do clean code e o que ele trouxe de benefícios pra quem realiza os códigos?"
"Quais os pilares de boas práticas da metodologia clean code?"
"Qual é o impacto da Regra do Escoteiro no dia a dia?"

Essas perguntas me ajudaram a explorar o tema, mas frequentemente retornavam respostas genéricas ou pouco aplicáveis.
Fui ajustando minha abordagem. Passei a estruturar melhor os prompts, trazendo mais contexto, direcionamento e intenção prática.

Em vez de apenas perguntar sobre os objetivos do Clean Code, comecei a solicitar organização em tópicos e aplicação no dia a dia
Ao explorar conceitos como funções pequenas ou TDD, passei a incluir impactos em manutenção, legibilidade e qualidade de software
Quando queria aprofundamento, adicionei pedidos por exemplos reais e cenários práticos

Esse processo de refinamento revelou algo importante:
Perceber que a IA responde exatamente ao nível da pergunta que recebe.

## Dificuldades encontradas:
Respostas amplas demais, sem profundidade
Explicações corretas, mas pouco aplicáveis na prática
Dificuldade em manter consistência entre diferentes respostas

Essas situações me fizeram perceber que o problema não estava na ferramenta, mas na forma como eu estava conduzindo a interação.

## Miniguia Estudos Clean Code:

Este miniguia foi construído a partir dos estudos realizados com apoio de IA, com base nos ensinamentos de Clean Code de Robert C. Martin.
A proposta é servir como um material de revisão rápida, reunindo os principais conceitos, definições e ferramentas práticas para aplicação no dia a dia.


# O que é Clean Code?
Clean Code é a prática de escrever código que seja fácil de ler, entender e manter. Não se trata apenas de fazer o código funcionar, mas de torná-lo claro para outros desenvolvedores (e para você mesmo no futuro).


# Principais objetivos

. Melhorar a legibilidade do código
. Facilitar manutenção e evolução do sistema
.Reduzir a incidência de bugs
. Tornar o código mais previsível e organizado


 # Boas práticas fundamentais

.Usar nomes claros e significativos
.Escrever funções pequenas e com responsabilidade única
.Evitar duplicação de código
.Manter o código simples
.Escrever comentários apenas quando necessário



# Regra do Escoteiro:

 “Deixe o código melhor do que você encontrou.”

Essa prática incentiva melhorias contínuas no código, mesmo em pequenas alterações do dia a dia.


# Funções pequenas:

Funções pequenas são mais fáceis de:

* Entender
* Testar
* Reutilizar
* Manter


# Testes e TDD segundo Robert C. Martin:

O autor defende que testes são parte essencial do desenvolvimento, não um complemento.

O TDD (Test Driven Development) propõe escrever testes antes do código, garantindo maior qualidade e segurança.


# Três leis do TDD:

1. Não escreva código de produção sem antes escrever um teste que falhe
2. Escreva apenas o suficiente do teste para falhar
3. Escreva apenas o código necessário para passar no teste



#  Glossário:

. **Clean Code**: Código limpo, legível e de fácil manutenção
. **Refatoração**: Melhorar o código sem alterar seu comportamento
. **TDD (Test Driven Development)**: Desenvolvimento orientado a testes
. **Legibilidade**: Facilidade de leitura e compreensão do código
. **Manutenibilidade**: Facilidade de modificar e evoluir o sistema
